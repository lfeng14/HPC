# Cache（高速缓存）简要解析
Cache 是**CPU 与内存之间的高速小容量存储器**，用来**缓存最近/最常使用的数据和指令**，让 CPU 尽量少访问慢速内存，从而大幅提升运行速度。

## 你给的两段关键含义
1. **加载/存储指令的顺序会影响缓存**
   数据怎么读、怎么写，会直接决定：
   - 哪些数据被放进缓存
   - 哪些旧数据被**替换（逐出）**出去
   顺序不合理，会造成**缓存命中率低**，速度变慢。

2. **预加载（Preload）可以提速，但乱用会减速**
   - 提前把要用的数据主动读到缓存里，能减少等待，**提升性能**。
   - 但如果预加载了**暂时不用**的数据，会把缓存里本来有用的数据挤出去，反而**降低性能**。

一句话总结：
**Cache 就是 CPU 的“临时快存”，指令顺序和预加载用得好就快，用不好就拖慢。**

这句话的核心意思可以**极简理解**为：

在 ARM 汇编与 C 混合调用时：
1. **函数参数的顺序非常关键**，因为它**直接决定了参数会被放在哪些寄存器里**。
2. 编译器和汇编代码都必须遵守一套统一约定，这套约定叫 **Procedure Call Standard（过程调用标准）**。
3. 你写汇编时，必须按这个标准去**对应寄存器取参数**，否则参数会取错。

---

### 用更通俗的话讲
- C 代码里写的函数参数是第1个、第2个、第3个……
- 到汇编里，它们就**固定对应**到 `x0`、`x1`、`x2`、`x3` 等寄存器。
- 顺序一变，寄存器就错了，函数直接跑飞。
- 大家都遵守的这套“谁放哪个寄存器”的规矩，就是 **过程调用标准**。
